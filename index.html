Meeting Date: 06 May 2025  ·  Attendees: Merlin Operational Head, ERP Team  ·  Subject: ERPNext v16 Workflow Updates & New Requirements
MOM Point 01
Action Required
In-House Manufacturing Module — BOM, Work Order, Job Card
Client requires own manufacturing capability for select Finished Goods (FG) items in addition to existing subcontracting.
  • What: Configure Manufacturing Module with BOM, Work Order, and Job Card for in-house FG production
  • Who: ERP Implementation Team + Merlin Production Planner
  • When: BOM setup before go-live; Work Order and Job Card tested in Phase 1 UAT
  • How: ERPNext Manufacturing Module — Item Master flag "Manufacture" or "Subcontract" to be set per item
  • Action: Share list of FG items to be manufactured in-house vs subcontracted
MOM Point 02
Action Required
Auto Order Sync from eCommerce Channels
Client wants automatic Sales Order creation from Amazon, Flipkart, Meesho, Shopify, Zepto, and Blinkit into ERPNext without manual entry.
  • What: Multi-channel order ingestion with auto SO creation, channel tagging, warehouse allocation, and logistics partner assignment
  • Who: Tech Integration Team (Frappe Developer) + Channel Managers
  • When: Integration spec to be finalised within 2 weeks; go-live in Phase 2
  • How: Custom Frappe App or third-party connector (Unicommerce / Vinculum / Eshopbox) + Pincode-to-warehouse mapping table + courier serviceability API
  • Action: Client to share API credentials / seller panel access for each platform
  • Action: Define pincode-to-warehouse and channel-to-courier mapping rules
MOM Point 03
Action Required
Real-Time Stock Balance Sync to eCommerce Platforms
Client needs live inventory visibility on every eCommerce platform at the individual item level to avoid overselling or stockouts.
  • What: Push available stock qty per item from ERPNext to Amazon, Flipkart, Meesho, Shopify, Zepto, Blinkit in real time or on scheduled basis
  • Who: Tech Team (Integration Developer) + Inventory Manager
  • When: Configure after warehouse structure is finalised; UAT before go-live
  • How: Frappe Webhook / Scheduler → Platform Inventory APIs · Safety stock buffer per channel to be defined
  • Action: Client to confirm safety stock threshold per channel and auto-deactivation rule when stock = 0
MOM Point 04
Action Required
Combo / Bundle Order Management
Client sells combo SKUs (e.g., Bag + Bottle + Lunch Box as one listing) on platforms but manages inventory per individual component in ERP.
  • What: Product Bundle DocType in ERPNext to map combo SKU → individual items
  • Who: ERP Team + Catalog Manager
  • When: Bundle setup as part of item master configuration in Phase 1
  • How: ERPNext Product Bundle DocType · Combo availability = minimum available stock among all components · Platform push uses composite qty
  • Action: Client to share full list of combo SKUs and their component mapping
MOM Point 05
Pending Confirmation
POD — Proof of Delivery for B2B Customers
Client requires documented proof of delivery for all B2B (offline distributor / bulk buyer) orders with document attachment capability in ERP.
  • What: Mandatory POD document attachment in Delivery Note for B2B channel — LR copy, E-way bill, signed receipt, photo evidence
  • Who: Dispatch Team (scanning/uploading docs) + Accounts Team
  • When: At time of delivery confirmation before closing Delivery Note
  • How: ERPNext Delivery Note → Attach Files · Validation script to enforce mandatory attachment for B2B channel tag
  • Action: Confirm which document types are mandatory vs optional for B2B POD
MOM Point 06
Action Required
Multiple Warehouse Configuration
Client operates or plans to operate from multiple warehouse locations. Each warehouse needs independent stock ledger, allocation rules, and reporting.
  • What: Configure multi-warehouse hierarchy — Main WH (per city/zone), BIN WH, Fix WH, Rejected WH, Virtual Subcontractor WH, Dark Store WH (Q-commerce)
  • Who: ERP Team + Warehouse Manager
  • When: Warehouse master setup in Phase 1 before any stock transactions
  • How: ERPNext Warehouse DocType · Each warehouse mapped to company · Inter-WH transfer via Stock Entry
  • Action: Client to share complete list of physical warehouse locations with city, type, and channel mapping
MOM Point 07
Action Required
Clickpost Integration — Delivery Tracking & Status Masking
Client wants a unified delivery tracking system across all courier partners with normalised status labels (delivery mask) visible in ERPNext and to customers.
  • What: Clickpost API integration — AWB push, status webhook back to ERPNext, unified tracking dashboard, RTO auto-matching to Sales Return
  • Who: Tech Integration Team + Ops Team
  • When: Integration in Phase 2 alongside logistics auto-allocation setup
  • How: Clickpost REST API → Frappe Custom App · AWB field in Delivery Note · Status field updated via webhook · Delivery mask table (courier status → Merlin status)
  • Action: Client to share Clickpost account credentials or initiate account setup
  • Action: Define standard delivery status labels for customer-facing communication
